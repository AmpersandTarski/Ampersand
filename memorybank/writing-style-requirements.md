# Writing Style Requirements

**Date:** July 31, 2025  
**Context:** Data migration research report NVWA FC4

## Language Usage Requirements
### Facts
 - Strictly separate opinions and facts, so the reader can clearly see the distinction.
 - Always substantiate traceable facts, so the reader can trace the source to check the fact.
 - Avoid non-traceable facts, to prevent even the appearance of hallucination.
 - Only refer to sources that actually exist. Always check URLs to verify they actually exist.

### Style
- Solid style with simple language use
- Radiate objectivity without explicitly mentioning it
- No sales-style
- Only use bullet lists or enumerations when strictly necessary. Otherwise just make a paragraph with proper sentences.
- Complete good, grammatically correct sentences. So no telegram style.

### Grammar and Word Choice  
- No adverbs or adjectives unless essential for meaning. Avoid adjectives and adverbs that can be left out without altering the meaning of the sentence. Especially suspect are words such as fundamental, comprehensive, important, significant, very, etcetera because these are words sales people use all the time.
- Active language use, so no passive voice.
- Short sentences

### Examples

**Avoid:**
- "significant amount of recognizable structure", because this is not concrete.
- "The proof that this structure can be successfully automated", because this is passive voice.
- "strong substantiation", because "substantiation" is already sufficient.

**Use:**
- "structure"
- "This structure is foundation"
- "This structure is automatable" 
- "substantiation"

### Application
Use this style for all texts you generate.
